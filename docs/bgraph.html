getRoad()

Returns an object with an array ('road') containing the current roadmatix (latest edited version), as well as the following members:
  • valid: whether the edited road intersects the pink region or not
  • loser: whether the edited road results in a derailed goal or not
  • asof: unix timestamp for "now"
  • horizon: unix timestamp for the current akrasia horizon
  • siru: seconds in rate units

Zooms out the goal graph to make the entire range from tini to tfin visible, with additional slack before and after to facilitate adding new knots.

Brings the zoom level to include the range from tini to slightly beyond the akrasia horizon. This is expected to be consistent with beebrain generated graphs.

<inner> adjustYScale()

Adjust the scale and range for the Y axis based on the current range of the Y axis. The Y axis range depends on the graph configuration, including whether it's a headless graph for a screenshot, an interactive graph or the editor.

<inner> brushed()

Gets called by d3.brush whenever the brush on the context graph is modified by the user

<inner> computeBoxes()

Recomputes padding value and bounding boxes for various components in the graph. In particular, plotpad, contextpad, plotbox and contextbox are computed.

<inner> computeXTicks()

Compute locations and labels for X axis ticks corresponding to the entire graph range for different zoom levels. These are stored in the "ticks" member of the bgraph instance. utilized later by the redrawXTicks() function for rendering.

<inner> createGraph()

Creates all SVG graph components if a graph DIV is provided. Called once when the bgraph object is created.

<inner> createTable()

Creates all road matrix table components if a table DIV is provided. Called once when the bgraph object is created.

Redraws X Axis tick marks based on the current X axis range for the focus graph, making "smart" decisions on what type of ticks to use. Tick mark types are precomputed and stored in the "ticks" member by the computeXTicks() function
